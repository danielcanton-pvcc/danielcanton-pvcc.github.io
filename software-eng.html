Chapter 7: Software Engineering
TERM DEFINITION
Software Engineering The subject of software development
Integrated Development Environment (IDE) Combines tools for developing software into a single, integrated package
Association for Computing Machinery
  1. Purpose of ACM
  2. Benefits of ACM Membership
  3. Student ACM Membership
  1. Brings together educators, researchers, and professionals to share resources and address challenges
  2. Networking opportunities, many subscriptions, special interest group availability, volunteering opportunities
  3. Mostly the same benefits as Professionals, but without the ability to vote for ACM officers
Institute of Electrical and Electronics Engineers (IEEE)
  1. Purpose of IEEE
  2. Benefits of IEEE
  3. Student IEEE Membership
  1. Organization dedicated to advancing technology for the benefit of humanity
  2. Networking, create a professional profile, join discussions on technology, create a group for projects
  3. Career guidance, student awards and scholarships
Software Life Cycle: Development Phase
  1. Requirements Analysis
  2. Design
  3. Implementation
  4. Testing
  1. To specify what services the proposed system will provide, identify conditions, and define outside interaction
  2. Creates a plan for the construction of the proposed system
  3. Actual writing of programs, creation of data files, and developing databases.
  4. Tests the results of each step of the development process for accuracy
Modular Implementation The process of developing and integrating modules
Unified Modeling Language (UML) Modern collection of tools that was developed for the object-oriented paradigm
Use Case Interactions between the system and its users
Use Case Diagram A diagram that depicts the proposed system with interactions being ovals and users being stick figures
Data Relationship
  1. One-to-one
  2. One-to-many
  3. Many-to-many
  1. Patients and occupied private rooms with 1 patient per room
  2. Physicians and patients with multiple patients per physician
  3. consulting physicians, physicians and patients with each physician having multiple patients and vice-versa
Design Pattern A predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) A group charged with overseeing and enforcing quality control systems used by the organization
Software Testing:
  1. Glass box testing
  2. Black box testing
  3. Beta testing
  4. Alpha testing
  1. The software tester is aware of the interior structure of the software
  2. Software testing that does not rely on interior knowledge of the software
  3. Preliminary version of the software is given to a segment of intended audience to test how it will perform in real situations
  4. Same as Beta Testing but at the Developers site instead of a public audience
Documentation:
  1. User documentation
  2. System documentation
  3. Technical documentation
  1. Explains the features of the software and describes how to use them
  2. Describes the softwares internals for easier maintenance
  3. Describes how the software should be installed and serviced
Intellectual Property Law
  1. Software license
  2. Software disclaimer
  3. Best defenses against software liability claims
  1. Legal agreement between the owner and user of a software product
  2. Included in the license to state the limit of the developers liability
  3. Apply sound software engineering principles during development, use a level of care compatible with the application, and maintain good records that validate these efforts
