All About GitHub
  • Q. What is a version control system?
    • A. Systems responsible for managing changes.

  • Q. What is Git?
    • A. Git is the Version Control System used by GitHub.

  • Q. What is GitHub?
    • A. GitHub is a platform that hosts repositories for users and developers.

  • Q. What is the difference between Git and GitHub?
    • A. Git is the system used to track changes within individual projects. GitHub is the website that hosts the system.

  • Q. Who started GitHub and how was it started?
    • A. Tom Preston-Werner, Chris Wanstrath, P.J. Hyett, and Scott Chacon launched GitHub publicly in 2008. It was originally created to help developers collab on large projects.

  • Q. What company owns it now?
    • A. Microsoft currently owns GitHub, but allows it to run indepedently.

  • Q. How much does a GitHub account cost?
    • A. GitHub has 3 tiers of subscription.
        Free for basic usage
        4$ Per user for teams and organizations for more storage and support
        21$ Per user for everything the Team tier offers, plus more security, support, and benefits

  • Q. What is the Octocat?
    • A. Octocat is the mascot of GitHub, designed as clip art by Simon Ocley and adopted for use by GitHub.
  • Git/GitHub Terms:
    • Repository: A place to store the code and files of a project.
    • Commit: Records changes to one or more files
    • Fork: A new repository that shares code with the original
    • Push: A command to send local repository files to a remote repository
    • Pull Requests: A command to download content from a report repository to the local one
    • Workflows: An automated process of one or more jobs
    • Issues: Tracks feedback, ideas, tasks and bugs
    • Raw Button: Allows a user to view the content of a file without any stylings
    • Blame Button: Used to find the last author who made each modification in a file
